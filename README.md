
# 🔐 Suricata IDS/IPS Lab – Network Threat Detection

## 📌 Project Description

This project focuses on implementing and configuring an Intrusion Detection System (IDS) using Suricata in a virtualized environment to analyze network traffic and detect simulated malicious activities.

The lab is oriented toward Blue Team operations, including security monitoring, alert analysis, and threat detection.

---

## 🎯 Objectives

* Deploy Suricata as an IDS on Linux
* Monitor and analyze network traffic
* Detect simulated attack scenarios
* Create and tune custom detection rules
* Reduce false positives through rule optimization

---

## 🧪 Lab Environment

* Virtualization: VirtualBox
* IDS System: Linux (Debian/Kali)
* Attacker Machine: Kali Linux

### Tools Used:

* Suricata
* Wireshark
* tcpdump
* Nmap

---

## 🌐 Network Topology

[Attacker] ⇄ [Internal Network] ⇄ [Suricata IDS]

* The IDS monitors traffic between virtual machines
* Attacks are simulated from the attacker machine

---

## ⚔️ Simulated Scenarios

### 1. Port Scanning (Nmap)

Performed network scanning to identify open ports.

**Result:**

* Detection of multiple SYN packets
* Alerts generated by Suricata

---

### 2. Brute Force Simulation

Multiple failed authentication attempts were generated.

**Result:**

* Detection of repetitive patterns
* Suspicious activity alerts triggered

---

## 🔍 Traffic Analysis

Traffic was analyzed using Wireshark and tcpdump to:

* Inspect TCP/IP packets
* Identify TCP handshake (SYN, SYN-ACK, ACK)
* Detect abnormal patterns

---

## ⚙️ Suricata Configuration

* Enabled default rule sets
* Created custom rules for:

  * Port scanning detection
  * Suspicious traffic
* Applied threshold tuning to reduce alert noise

---

## 📊 Results

* Successful detection of simulated attacks
* Real-time alert generation
* Reduction of false positives
* Improved understanding of network behavior

## 🚧 Challenges
